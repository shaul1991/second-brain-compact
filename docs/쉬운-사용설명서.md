# 세컨드 브레인 쉬운 사용 설명서 (비개발자용)

이 설명서는 프로그래밍을 몰라도 이 도구를 쓸 수 있도록 쉬운 말로 적었습니다.
어려운 용어는 일부러 피했어요.

---

## 1. 이게 뭔가요? (1분 설명)

당신의 **개인 메모 캐비닛**이라고 생각하세요. 메모를 적어 보관하고, 나중에 꺼내 보는
도구입니다. 두 가지가 특별합니다:

- **전부 내 컴퓨터 안에만 있습니다.** 인터넷의 남의 회사 서버로 메모가 나가지 않아요.
- **내가 주인입니다.** 메모는 평범한 글자 파일로 저장돼서, 이 도구가 사라져도 메모는
  그대로 남습니다.

여기에 **똑똑한 비서(AI)** 를 연결할 수 있습니다. 비서는 메모를 찾아주고 정리를 도와주지만,
**무엇을 정식으로 남길지는 항상 당신이 결정**합니다.

---

## 2. 꼭 알아야 할 3가지 개념

### ① 임시 메모
뭔가 적으면 일단 **"임시" 서랍**에 들어갑니다. 아직 정식이 아니에요.

### ② 도장 찍기 (정식으로 만들기)
임시 메모를 보고 "이건 쓸 만하다" 싶으면 **도장을 찍어 정식 노트로 승격**시킵니다.

### ③ 숨김 규칙 (가장 중요!)
**도장을 찍기 전까지 임시 메모는 검색에 안 나옵니다.** 일부러 그렇게 막아둔 거예요.
정리 안 된 메모가 결과를 어지럽히지 않도록요. 그리고 **도장은 오직 당신만** 찍을 수
있습니다. 비서(AI)가 멋대로 정식으로 만들지 못해요.

> 한 줄 요약: **적기 → (내가) 도장 → 검색 가능**

---

## 3. 처음 한 번만: 설치하기

솔직히 말씀드리면, **처음 설치만큼은 약간의 준비**가 필요합니다. 한 번만 하면 돼요.

1. 컴퓨터에 **Docker** 라는 무료 프로그램을 설치합니다.
   (인터넷에서 "Docker Desktop 설치"를 검색해 따라 하면 됩니다. 어려우면 주변
   개발자에게 이 단계만 한 번 부탁하세요.)
2. 이 도구 폴더를 연 다음, 아래를 그대로 복사해 붙여넣고 실행합니다:

   ```bash
   cp .env.example .env
   docker compose build
   docker compose run --rm cli init
   ```

이게 끝나면 `data/` 라는 폴더가 생깁니다 — 여기에 당신 메모가 글자 파일로 쌓입니다.

> 한 번 설치한 뒤로는 아래 "매일 쓰는 법"만 보면 됩니다.

## 4. 매일 쓰는 법

### 가장 쉬운 길: 비서(AI)에게 말로 시키기

Claude·Codex·Gemini 같은 AI에 이 캐비닛을 연결해두면(아래 6번), **그냥 말로** 하면 됩니다:

- **적기**: "이거 기억해둬: 다음 회의는 화요일 3시" → 비서가 임시 메모로 저장합니다.
- **찾기**: "회의 시간 적어둔 거 뭐였지?" → 비서가 정식 노트에서 찾아줍니다.
- **읽기**: "그 노트 전문 보여줘" → 비서가 내용을 보여줍니다.

비서가 할 수 있는 건 **적기 · 찾기 · 읽기 · 현황보기** 네 가지뿐입니다.
**도장 찍기와 버리기는 당신만** 할 수 있어요(아래 명령으로).

### 직접 명령으로 하기

비서 없이 직접 할 수도 있습니다. 폴더에서 아래 명령을 씁니다.

**메모 적기**
```bash
docker compose run --rm cli ingest --json '{"body":"여기에 메모 내용","title":"제목","tags":["태그"]}'
```
실행하면 `CAP-local-날짜-0001` 같은 **메모 번호**가 화면에 나옵니다. 이 번호를 도장 찍을 때 씁니다.

**도장 찍기 (정식 노트로 승격)**
```bash
docker compose run --rm cli promote --id CAP-local-날짜-0001 --target notes
```
이제 이 메모가 검색에 잡히는 정식 노트가 됩니다.

**안 쓸 메모 버리기**
```bash
docker compose run --rm cli archive --id CAP-local-날짜-0001 --reason "필요 없음"
```
(완전히 지우는 게 아니라 "버림 서랍"으로 옮겨, 나중에도 흔적이 남습니다.)

**현황 보기 (메모 몇 개 있나)**
```bash
docker compose run --rm cli status
```

## 5. 똑똑한 검색 켜기 (선택 사항)

기본 검색은 **글자가 똑같아야** 찾습니다. (예: "사과"라고 적었으면 "사과"로 검색)

**똑똑한 검색**을 켜면 **뜻으로** 찾아줍니다. (예: "과일 적어둔 거?" 라고 물어도 "사과"
메모를 찾아줌)

- 켜는 법: `.env` 파일에서 `SEMANTIC_SEARCH_ENABLED=true` 로 바꾸고, 아래를 실행합니다.
  ```bash
  docker compose --profile semantic up -d supermemory ollama
  docker compose run --rm cli reindex
  ```
- 이 똑똑한 검색도 **전부 당신 컴퓨터 안에서** 돌아갑니다. 인터넷으로 안 나가요.
- 켜지 않으면(기본) 평소처럼 글자 검색으로 잘 돌아갑니다. 부담되면 안 켜도 됩니다.
- 메모를 직접 손으로 고쳤거나 새 컴퓨터로 옮겼다면 `reindex` 를 한 번 실행해 검색을
  최신으로 맞춥니다.

> 참고: 이 똑똑한 검색 기능은 실제 환경에서 한 번 더 점검이 필요한 부분이 있습니다.
> 켜기 전에 주변 개발자에게 한 번 확인받으면 안전합니다.

## 6. 백업하기 / 새 컴퓨터로 옮기기

메모를 안전하게 보관하고 다른 컴퓨터로 옮기는 방법입니다. **당신만의 비공개 저장소**에
사진 찍듯 백업합니다 (남이 못 봅니다).

```bash
docker compose run --rm cli data sync
```

새 컴퓨터에서는 그 저장소에서 메모를 받아오면 됩니다. (똑똑한 검색을 쓴다면, 받아온 뒤
`reindex` 한 번만 실행하면 검색도 다시 만들어집니다.)

## 8. 자주 묻는 질문

**Q. 내 메모가 인터넷으로 새나가나요?**
아니요. 전부 당신 컴퓨터 안에서만 돌아갑니다. 똑똑한 검색을 켜도 마찬가지예요
(인터넷 주소로는 연결을 아예 거부합니다).

**Q. AI가 멋대로 내 정식 노트를 만들거나 지우나요?**
아니요. 도장 찍기(정식화)와 버리기는 **당신만** 할 수 있습니다. AI는 적기·찾기·읽기만 합니다.

**Q. 이 프로그램을 지우면 메모도 사라지나요?**
아니요. 메모는 평범한 글자 파일로 `data/` 폴더에 남습니다. 메모장으로도 열 수 있어요.
(Obsidian 같은 메모 앱으로 그 폴더를 열어 보기 좋게 볼 수도 있습니다.)

**Q. 임시 메모를 깜빡 검색했는데 안 나와요. 고장인가요?**
정상입니다. 임시 메모는 **도장을 찍어야** 검색에 나옵니다(2번 ③ 참고).
